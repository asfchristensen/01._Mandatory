Variables



Variables are containers for storing values. There exist two types: local variables (inside functions with local scopes) and global variables (outside functions with global scopes). There are some rules while declaring a JavaScript variable (known as identifiers/name): it must start with a letter (a/A-z/Z), underscore(_) or dollar sign($).


Declaration keywords


Dependening on the keyword, there are different scopes.
You can declare variables in JavaScript in four ways:

1) const
2) let
3) var
4) Nothing


const

- Not a constant!
- Always use it if you can get away with it
- Has to be declared in the declaration line
- Block scoped (can't be accessed from outside the block)
- Must assign a value when declaring with const

    
        const me = { name: "Name" };

- Possible to assign a property to the object

    
        const hobbies = ["Hobby1", "Hobby2"];
        hobbies.push("Hobby3");
        me.hobbies = hobbies;

- Cannot be reassigned / change value after initialization

    
        const me = { age: 30 };
        me = { }; // not allowed!

- Cannot be re-declared

    
        const car = "Volvo";
        const car; // not allowed!
let

- Use it when you need it
- Use it if the value is going to change e.g. scores
- Block scoped (can't be accessed from outside the block)

    
        let value = true;
        {
        let value = false;
        }
        console.log(value); // true
    

    
        for (let i = 0; i <= 5; i++) {
            setTimeout( () => {
                console.log(i); // 0,1,2,3,4
            }, 1000);
        }

- Cannot be re-declared

    
        let car = "Volvo";
        let car; // not allowed!
var

- Never use this!!!
- No scope (can be accessed from everywhere)


Global variables


Be careful when using global variables. First of all, if you declare a global variabel and accidently re-declare it, the first variable will be overwritten. Secondly, global variables take longer for JavaScript to find than local ones. Eventhough the speed difference isn't huge, it still exists. Thirdly it can cause maintenance difficulties due to increased coupling that reduces testability in the code.
    
        "use strict";

        totalGlobalVariable = "Never ever do this";
        var globalVariable = "Also never do this";


        var value = true;
        {
            var value = false;
        }
        console.log(value); // false


        for (var i = 0; i <= 5; i++) {
            setTimeout( () => {
                console.log(i); // 6,6,6,6,6,6
            }, 1000);
        }

The 'i' is global do to 'var' and therefor 'i' points at the global 'i' that after 1 second already incremented to 6, and prints it out 6 times.
