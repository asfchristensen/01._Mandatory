Nodemon

A command-line utility for Node.js that helps in the development process by automatically restarting the Node.js application, whenever changes are made to the source code or the filesystem. This saves a lot of time when working on Node.js projects, because you would otherwise have to manually stop and restart the application every time a change is made in the project, by constantly typing the command: node filename.js.

The tool is installed globally with npm and can be used with any Node.js project. It also provides several configuration options that can be used to customize the behaviour of the tool e.g. ignoring specific files, specifying a delay before restarting, execute and restart HTML and JavaScript files eventhough they are to be read with .readFileSync() etc.

Eventhough the tool speeds up the process for development, there are downsides too:

- Increased ressource usage (it monitors the file system for changes which can increase CPU and memory usage, and it can slow down the computer when working on large projects)

- False positives (it can detect not-relevant changes that the application doesn't use and cause unnecessary restarts that can slow down the process)



Postman

This tool is used for testing APIs without a browser. It has an interface for making HTTP requests to API endpoints and recieving responses, which makes it easy to test and see, if the backend of the project works. It is possible to save the requests , so you don't have to write them every time, which can make the process easier and faster.


Deployment


Vercel

A cloud platform for building and deploying static websites, and it supports the Node.js framework. It is easy to use and free of charge (for small projects), and provides integration with GitHub.

One of the downsides encountered by using this tool is that you have to embed the CSS and JavaScript in the HTML file, or else Vercel can't display the styling or dynamic functionality. This means you can't have a package and file structure in your project.

If this tool is used, you have to create a "vercel.json" file in the root of the project and write:
    
        {
            "version": 2,
            "builds": [
                {
                    "src": "app.js",
                    "use": "@now/node"
                }
            ],
            "routes": [
                {
                    "src": "/(.*)",
                    "dest": "app.js"
                }
            ]
        }

Clean up code


Prettier

A code formatter that helps clean up code. It points out where the mistakes are and enforces a consistent style, which makes the code easier to read and maintain. It can be installed as a plug-in/extension in VSC.


ESLint

A linter that checks code for syntax errors, coding standards and other potential issues. Can be used to enforce coding conventions, identify code that is difficult to maintain or prone to errors. It can be installed as a plug-in/extension in VSC.
