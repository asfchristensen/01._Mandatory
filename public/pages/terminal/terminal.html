Terminal commands



Why use the terminal in Node.js?


- It's more universal and not bound to an IDE
- No GUI when working with servers e.g. intelliJ


Git commands


Clone project from GitHub

- Create a repo on GitHub
- Copy the URL
- In the terminal navigate to the folder with "ls" and "cd" where you want to store the project
- Clone the project in that folder


user@user 4._semester % ls
Python Node.js
user@user 4._semester % cd Node.js
user@user Node.js % git clone https://github.com/annesofiefogh/Node.js_2023
Cloning into 'Node.js_2023'...


Commit and push code to GitHub

- Navigate to the working directory/folder
- From here write three commmands
- The first command can either be with '.' or "-A" ("all files in the directory")


user@user Node.js_2023 % cd 06._Upper
user@user 06._Upper % git add .
user@user 06._Upper % git commit -m "Commit message here.."
...changes...
user@user 06._Upper % git push



Node commands


Helping command

- See the version of node running on your computer and see how to get help
- Gets into a prompt '>' where you can write node commands
- E.g. "typeof" command that shows the datatype you are working with
- To exit the prompt, press 'Ctrl + C' (soft quit) or 'Ctrl + D' (hard quit)


user@user Node.js_2023 % node
Welcome to Node.js v18.14.0.
Type ".help" for more information.
>
> typeof 2 + "12"
> "212"


Install packages

- When you want to use libraries in your node project you have to install packages
- First install "node package manager" (npm) before any packages can be installed
- When you run this command, it will automatically set up the "node_modules" directory (greyed out because it's placed in .gitignore)
- To make the install" you can either write "i" or "install"


user@user 06._Upper % npm install
...installing the node package manager...


- When you want to install a specific package, after having created the package.json file in the root of the project
- Write "sudo" before it can install the packages (avoids the EACCES error)
- The packagename could be "express" or "nodemon" etc.
- Use the package across the entire application, add the flag "-g" (globally)


user@user 06._Upper % sudo npm install packagename
...installing a the package/library...

user@user 06._Upper % sudo npm install nodemon -g
...installing nodemon so it can be run in every file in the aplication...


Running files with node

- If you want to test if your code works
- Every time you make a change in your project, you have to type this command


user@user 06._Upper % node "filename.js"
...code running...


- If you want to run the scripts automatically with automatic restart
- To stop running nodemon, type: 'Ctrl + C'


user@user 06._Upper % nodemon "filename.js"
[nodemon] 2.0.21
[nodemon] to restart at any time, enter `rs`
[nodemon] watch path(s): *.*
[nodemon] starting `filename.js`


- If used fs.readFileSync() in the application and written the nodemon.json file
- This command ensures nodemon still updates files in the application dynamically
- The filename.js is typically "app.js" with the routes

user@user 06._Upper % nodemon --ext html,js filename.js


Initialize a new Node.js project

- If you want to initialize a new Node.js project and thus also create your own "package.json" file automatically
- Add the flag "-y" to the command to avoid being prompted to answer a series of questions about the project
- If you want to deploy the project the packagename has to bee unique
- To get the package into the project after this, write: npm i pacakgename.


user@user 06._Upper % npm init -y
